My skills as a developer

Skills

I have a good grounding in Java from university. We covered interfaces and abstract classes, errors and debugging as I learned the basics of the language. We wrote simple programs to calculate basic statistics, and large programs with multiple classes and interfaces.

I hadn't revisited Java in the last year until the interview. Alas I had forgotten a great deal of it, but it is far quicker and easier to relearn than to learn the first time, and that I have been throwing myself into, in the hope you would give me another chance to present myself for interview at 67 Bricks.



Before last week I'd never heard of Bootstrap. I have since built this whole website on the framework, validated all the html at validator.w3.org and even got the MySQL database connected to php, see here.

I have encountered many problems while building this website, banged my head against brick walls (probably not 67 of them), Googled extensively, joined GitHub, rejoiced in the victories and finally I have a website, all bugs ironed out.

To say this has been a learning curve is an understatement. I've learned so much more about html, css, MySQL and php in this practical activity than I have in the whole time at university. I appreciate the theoretical grounding uni has provided, but I learn by doing not reading. Imagine how quickly I could learn to code at your company, especially with learned help to assist with the brick walls.
